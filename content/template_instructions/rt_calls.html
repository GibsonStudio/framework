RT Calls are created as javascript objects and then added to the sequence with the function add_rt_call(object);. Here is a quick refence list of rt call object properties - (type) [default value]

  • type (string) ['plane'] - optional
  • id (string) [''] - required
  • audio (string) [(rt.id).mp3] - optional
  • text (string) [''] - required
  • position (string) ['left'] - optional
  • image (string) [rt_(rt.type)_icon.png] - optional
  • onstart (string) - optional
  • onended (string) - optional

Now to explain these in more detail:

type (string)

values: plane, tower, ground

Sets the type of unit making the call. Also defines the colour of the RT text and the image to use. If rt.image is not set, the image rt_type_icon.png will be used. e.g. If rt.type = 'tower', the image rt_tower_icon.png will be used.

id (string)

Sets the id of the RT Call. This is primarily used to identify the text box for the rt text to appear in, but if rt.audio is NOT set, rt.id also defines the audio file for the rt call:

audio (string)

Sets the audio file for the RT Call. Note: Do not include the file extension.

Audio files must be .mp3 and be in the same location as the HTML file.

text (string)

This is the text to display when the RT cll is played:

The text will be automatically coloured depending on the value oy rt.type:

  • plane: blue (#3e4899)
  • tower: green (#3fa535)
position (string)

values: left, right

Whether the RT call is positioned to the left or the right of the RT calls container.

image (string)

The image to use for the RT call icon:

Images should be in root/images/rt/ (so they can be used by multiple lessons) and should have a height of 80px.
