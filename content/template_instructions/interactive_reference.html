Note: The following are used for drag and drop interactivity. This is explained in detail here: drag_and_drop.html

add_object (object)
  • object: Object. A drag and drop object.
dragstart (e)
  • e: Event. A drag and drop object.

dragend (e)
  • e: Event. A drag and drop object.

This function and the event are automatically set up by the function add_object:

drop (e)
  • e: Event. A drag and drop object.

This function and the event are automatically set up by the function add_object:

dragleave (e)
  • e: Event. A drag and drop object.

This function and the event are automatically set up by the function add_object:

dragover (e)
  • e: Event. A drag and drop object.

This function and the event are automatically set up by the function add_object:

add_drag_drop_ui (include_output)
  • include_output: Boolean (0 or 1). Defines whether or not to include an output div for user feedback.

Adds a submit button and an output div (if include_output == 1).

drag_drop_submit ()

Called when the drag and drop submit button is clicked. Processes objects / targets and gives user feedback.

get_object_index_from_id (id)
  • id: String. Id of object.

returns: integer

get_object_position (id)
  • id: String. Id of div containing object.

returns: string

get_object_index_in_position (position)
  • position: String. Id of div (postion).
swap_object_positions (position1, position2)
  • position1: String. Id of div (postion).
  • position2: String. Id of div (postion).

This swaps the object.position property of the objects in position1 and position2 - it does not swap the HTML, i.e. swap the actual object positions on screen.

debug_objects ()

Outputs object debug info to the debug output div - this needs to be included with write_debug();.

Note: The following is used for 'choose the correct words' interactivity. This is explained in detail here: choose_words.html

check_select (selectid, correct_val)
  • selectid: String. Id of select input.
  • correct_val: String. Correct answer for select input.
